Using the Configuration

By Jonathan Peterson October 10, 2022

Several aspects of the generator can be changed and configured through the build/site-config.json. 

{
  "site": {
    "root": "https://eonasdan.github.io",
    "subfolder" : "static-site-generator"
  },
  "output" : {
    "main": "site",
    "posts" : "posts"
  },
  "source": "site-source",
  "server" : {
    "port": 62296
  },
  "defaultAuthor": {
    "name": "Fred Stone",
    "url": "https://eonasdan.github.io/static-site-generator/",
    "avatar": "https://placekitten.com/120/120",
    "bio": "Check me out. I do stuff."
  },
  "services": {
    "languageTools": true,
    "azureSearch": true
  }
}

Site

Root

This defines the base URL for your site. Generated links will use this value when creating fully qualified links e.g. https://eonasdan.github.io + page.html

Subfolder

This property is optional but when set, allows for hosting under a subfolder. If you host this one GitHub Pages without a custom domain, you should set the value to your project name. Note that for development, your project does not need to be under this folder. The development service will automatically handle that.

Output

Main

This is where the generated site will go. This is also the folder that gets served when you run NPM start.

Posts

This is where the generated posts will go. This is a subfolder under "main".

Source

The directory where the build scripts should look for the templates and partials.

Server

Port

The development server will use this port to host your site locally.

Default Author

These values are automatically loaded into the editor so that you don't have to type them in each time.

Services

These are optional external services that can enhance your local experience during page creation or your users experience after the site is deployed.

LanguageTool

You must have the docker container running for this flag to work. If you don't want to use LT for spelling and grammar, be sure this is set to false.

Azure Search

If this feature is turned on, the build process will generate an Azure Cognitive Search index document and allows the search box to use ACS.
