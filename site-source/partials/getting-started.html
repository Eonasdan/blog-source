Getting Started

By Jonathan Peterson October 9, 2022

Why?

I wanted a configurable, easy to use static site generator for my blog. I wanted to be able to write real HTML and not use markup to create content. At the same time, I didn't want to have to duplicate the layout and the usual headaches that come with HTML content.

I also wanted to have built in SEO and have things auto generate for me.

This project has grown to do a lot of things, including generating image sets at Bootstrap's default breakpoints. The built-in editor supports LanguageTool, which gives you near real time spelling and grammar checking.

It also supports adding an Azure Cognitive Search to let your visitors search your site with the power of Azure.

How?

This project makes heavy use of NodeJS to process partial pages and posts and combine them with template files. The pages are process to generate minimal CSS and JavaScript, along with sitemaps, manifests, image and SEO tags.

What?

The result is what I use for my own personal blog and I hope it's something that can work for you too.

Have A Tour Around

There's a lot of background stuff that you might not care about, so I will point out the most important pieces first if you want to use this for your own project.

Build

The build folder has a file called site.config.json. You will want to edit this file to fit your site. You can read more about that on this page .

Docker

If you want the LanguageTool integration, you will need to run languagetool.yml. This is a docker compose file and will set up everything needed to locally host the free version of LanguageTool. This is completely optional.

Editor

The editor folder contains the files for the WYSIWYG editor, powered by EditorJS. You can read more about this on this page.

Site source

This is the main folder for your site. If you want to change the look and feel then you can make changes to the template. You can read more about the templates here.

Temporary folders and Outputs

The generator will create an img_temp and ts-out folder. Both of these can be ignored and cleaned up. The img_temp folder hold any images that you upload through the editor. These files will automatically be deleted once you save the post or clear the editor.

The site folder is the completed output. You will want to publish this folder to your host or use a GitHub action to copy it to your gh-pages.
